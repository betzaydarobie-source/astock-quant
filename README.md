# astock-quant —— A股 中长期价值选股研究系统（学习/研究用）

一个**从零搭建、用于学习与研究**的 A股 量化系统。代码追求可读、可跑、可改 ——
**不是实盘交易系统，不构成任何投资建议。**

> **当前状态**：项目已从「预测短期涨跌」转向「中长期价值选股」。系统每季度挑出
> 一篮子「又便宜又能赚钱」的好公司，按季度持有，目标是长期对标沪深300指数。
> 详见 `人话报告/12-Stage5价值选股改造.md`（大白话版）。

---

## 这是什么

简单说：**这个系统帮你从 A股 里挑「又便宜、又能赚钱」的好公司，按季度更新一份
推荐名单。**

它不预测「明天哪只股票涨」——项目早期做过这件事，并用 8 天反复诚实地证明：用散户
能拿到的数据预测短期涨跌，**准确率约 51%，跟抛硬币没区别**。所以系统转了方向。

现在做的事，是有大量学术和实践证据支持的「价值投资 + 量化」：

- **挑便宜的**：看一家公司的市盈率（PE）、市净率（PB）跟它自己历史比，是不是处在
  便宜的位置。
- **挑能赚钱的**：看 ROE（用钱赚钱的能力）、净利率、毛利率这些质量指标。
- **挑在成长的**：看营收和利润的同比增速。
- **长期持有**：每个季度（约 60 个交易日）换一次仓，不做短线。

把这三类指标合成一个透明、可解释的「综合分」，每季度选综合分最高的 15-20 只。

### 为什么不在选股这层套机器学习

A股 财报一年只更新 4 次，4 年只有约 16 期数据 —— 样本量对机器学习是杯水车薪，硬套
只会过拟合噪音、回测漂亮实盘崩盘。价值投资的逻辑本身简单可解释，所以这一层用
**透明因子打分**：每一步都看得见，能直接告诉你「这只票为什么入选」（便宜？质量好？）。

## 价值选股思路

```
数据层（A股 行情 / 财务 / 估值历史分位）
        │
        ▼
因子打分（三个维度，每个维度由若干因子等权合成）
  ├─ 价值（便宜度）：PE↓  PB↓  股息率↑
  ├─ 质量（赚钱能力）：ROE↑  净利率↑  毛利率↑
  └─ 成长（增长性）：营收增速↑  净利增速↑
        │
        ▼
综合分 = 价值×0.4 + 质量×0.4 + 成长×0.2  （0~1，越高越「便宜+好」）
        │
        ▼
每季度初选综合分 Top 15-20，等权买入，持有整个季度
        │
        ▼
季度调仓回测：扣掉真实交易成本，和沪深300指数比超额收益
```

所有打分的标准化 / 排名都按「每日横截面」做（只用当天同一批股票互相比），这样
天然不会把未来信息泄漏进当前分数 —— 这是防 look-ahead 的根本。

## 回测结果（诚实版）

按季度调仓、对标真·沪深300指数，回测区间 2022-01 ~ 2026-04（约 4 年，18 次调仓）：

| 指标 | 价值选股策略 | 沪深300指数 |
|---|---:|---:|
| 累计收益 | **+20.73%** | **−7.96%** |
| 年化收益 | +4.54% | — |
| 年化超额收益（alpha） | **+6.25%** | — |
| 夏普比率 | 0.23 | — |
| 信息比率（IR） | 0.46 | — |
| 最大回撤 | −27.03% | — |

按年份分解（5 年赢 4 输 1，仅 2023 跑输；2026 为不完整年）：

| 年份 | 策略 | 沪深300 | 超额 |
|---|---:|---:|---:|
| 2022 | −7.01% | −21.27% | +14.27% |
| 2023 | −15.29% | −11.38% | −3.92% |
| 2024 | +31.06% | +14.68% | +16.37% |
| 2025 | +19.25% | +17.66% | +1.59% |
| 2026（不完整） | −1.93% | −2.24% | +0.31% |

### ⚠️ 必须一起看的诚实结论

这是一份**偏乐观的回测**，「回测没被证伪 + 有超额迹象」**不等于**「能在实盘赚钱」。
原因：

- **样本小**：只有约 4 年、单一市场环境。换一段历史（牛市/熊市/震荡市）结果可能
  完全不同。
- **偏乐观**：选股池若是「今日沪深300成分股」，自带幸存者偏差 —— 这 300 只是「涨
  上来、被指数纳入」的赢家，退市/被剔除的输家不在池子里，会让回测系统性偏好看。
  交易成本虽已计入，但用收盘价成交、未建模冲击成本，实盘成本通常更高。
- **超额主要靠 2024**：总超额很大程度是 2024 那一年堆出来的，不是每年稳定赢。
- **IR 偏低**：信息比率 0.46，说明超额收益相对其波动并不算高质量。

**正确口径**：这份回测最强只能给出「证伪」——若连偏乐观的回测都跑输指数，实盘几乎
必输。它跑赢了，只说明「方向值得继续研究」，**不足以支持实盘部署**。

> 这是一个**学习 / 研究项目，不构成任何投资建议。历史回测收益不预测未来。**

## 快速上手

### 0. 系统依赖（macOS 用户必读）

LightGBM 在 macOS 需要 OpenMP runtime（`libomp`）：

```bash
brew install libomp
```

否则 `import lightgbm` 会报「dyld: Library not loaded: ... libomp.dylib」。
Linux / Windows 通常自带，无需此步。

### 1. 安装 Python 依赖（用 uv 管理）

```bash
uv sync --extra dev
```

### 2. 跑价值选股季度调仓回测

这是当前系统的核心入口 —— 回答「价值选股能不能跑赢沪深300」：

```bash
# 默认（Top-15 持仓）
uv run python scripts/run_quarterly_backtest.py

# 换成 Top-20 持仓
uv run python scripts/run_quarterly_backtest.py --top-n 20

# 完整参数
uv run python scripts/run_quarterly_backtest.py --help
```

产出：

- `artifacts/quarterly_backtest/report_<date>.md` —— 人读版：策略 vs 沪深300 对照
- `artifacts/quarterly_backtest/results_<date>.json` —— 机读版：指标 + 按年超额

### 3. 跑测试

```bash
uv run pytest tests/ -v
uv run ruff check astock_quant/ tests/
```

### 4. 每日报告（可选）

系统也能生成每日 HTML 报告（推荐名单 + 回测成绩 + 实验性短期预测）。报告网站化
与 GitHub Pages 部署详见 `DEPLOY.md`。

## 项目结构

```
astock_quant/            主 Python 包
├── contracts.py         全局数据契约（模块间传递的 Pydantic models）
├── config/              配置（股票池、路径、回测参数、打分权重）
├── data/                数据层（A股 行情/财务适配器 + 缓存 + panel + 沪深300基准）
├── factors/             因子层
│   ├── value_score.py   ★ 价值+质量+成长 综合打分（当前选股核心）
│   ├── price_volume.py / fundamental.py / moneyflow.py   量价/财务/资金流因子
│   └── llm_factor.py    LLM 因子（把新闻转因子，历史保留）
├── labels/              标签层（含 value_ranking_label 季度排名标签）
├── models/              模型层（短期预测模型，已降级为实验性）
├── backtest/            回测层
│   ├── quarterly.py     ★ 季度调仓回测引擎（当前回测核心）
│   └── engine.py        逐日撮合引擎（T+1 / 涨跌停 / 成本 / A股 约束）
├── signals/             信号层（模型预测 → 可读信号）
├── predict/             每日报告生成（renderer + 模板）
└── pipeline/            端到端编排
scripts/                 命令行入口
├── run_quarterly_backtest.py   ★ 价值选股季度回测（当前主入口）
├── run_pipeline.py             短期预测链路（实验性，历史保留）
└── build_index.py              生成报告网站首页
tests/                   测试（覆盖 look-ahead / T+1 / 涨跌停 / 切分纪律 等）
人话报告/                大白话讲解报告（面向项目主人，当前有效）
dev-docs/                开发 / 设计文档存档（多为早期短期预测系统的设计）
docs/                    GitHub Pages 发布目录（报告网站）
```

> 项目早期围绕「短期涨跌预测」做过完整的数据/因子/模型/回测链路，那套代码（如
> `models/`、`scripts/run_pipeline.py`）**没有删除**，而是诚实地降级为「实验性、
> 已知接近随机」保留。这是项目的「加法纪律」：新方向加进来，旧的东西如实保留、
> 不偷偷隐藏。

## 诚信红线

这个项目把「不造假 alpha」当成铁律，三道防线一直没动：

1. **报告模板**：诚信声明永远放在报告最显眼的第一位，不可删、不可缩小。
2. **代码守门测试**：删掉诚信声明 → 自动化测试立刻报警，代码进不了主分支。
3. **如实标注**：早期那 4 个接近随机的短期预测模型，没删也没藏，如实标注「实验性、
   已知接近随机、仅供参考」。回测结果附免责声明，推荐名单附「不构成投资建议」。

> 量化投资里，99% 的「高收益策略」在回测里看起来很美，上实盘就翻车。这个项目选择
> 从一开始就把局限说清楚，宁可让数字不好看，也不造假。
